## Project Overview

BiSGuide is a World of Warcraft retail addon that displays Best-in-Slot gear for any class/spec across Raid, Mythic+, and PvP content types. It renders as a side panel attached to the Character Frame.

- **Interface version:** 120000 (WoW 12.0 — The War Within)
- **SavedVariables:** `BiSGuideDB`
- **No external Lua libraries** — uses WoW API directly (no Ace3, no LibDataBroker)

## Data Pipeline

BiS data is auto-generated by `scraper/scraper.py` (Playwright + Wowhead XML API) into `Data.lua`. Never hand-edit `Data.lua`.

```bash
# Install dependencies
pip install -r scraper/requirements.txt
playwright install chromium

# Regenerate all class/spec data
python scraper/scraper.py

# Single spec
python scraper/scraper.py --class mage --spec fire

# Preview without writing
python scraper/scraper.py --dry-run
```

Cached item/zone metadata lives in `scraper/item_cache.json`.

## Architecture

**Load order (defined in BiSGuide.toc):** `Data.lua` → `Core.lua` → `UI.lua`

All three files share state via the addon namespace: `local addonName, ns = ...`

| File | Role |
|------|------|
| **Data.lua** | Auto-generated lookup table `BiSGuideData[CLASS][Spec][contentType]` → array of item entries |
| **Core.lua** | Player detection (class/spec), event routing (`PLAYER_LOGIN`, `PLAYER_SPECIALIZATION_CHANGED`, `PLAYER_EQUIPMENT_CHANGED`), slash commands (`/bis`, `/bisguide`), accessor API (`ns:GetBiSList`, `ns:GetPlayerSpec`, `ns:SetViewClassSpec`, etc.) |
| **UI.lua** | Creates the panel frame anchored to `CharacterFrame`, dropdown menus for class/spec/content-type selection, scrollable item rows with tooltip integration and equipped-status indicators |

## Data Format

Each item entry in `Data.lua`:
```lua
{
    slot = 1,                        -- equipment slot ID (1–17, maps to ns.SLOTS)
    itemId = 249970,                 -- WoW item ID
    name = "Relentless Rider's Crown",
    source = "Tier Set (Relentless Rider's Lament)",
    quality = 4,                     -- 0=Poor … 5=Legendary
    ilvl = 289,
    bonus = "12806:13335"            -- colon-separated bonus IDs (used in hyperlinks)
}
```

Content types: `raid`, `mythicplus`, `pvp`.

## Key Patterns

- **Namespace methods** — public API uses `ns:FunctionName()`, private helpers are local functions.
- **Row pool** — UI.lua reuses frame objects via `rowPool[index]` to avoid frame churn.
- **Item links** — built with bonus IDs for correct tooltip ilvl: `"item:ID::::::::::::BONUS"`.
- **Class keys** — uppercase file names (`DEATHKNIGHT`, `DEMONHUNTER`); spec keys are English display names (`Blood`, `Havoc`).
- **Equipped check** — compares `GetInventoryItemID(slotID)` against BiS `itemId`; shown as green "E" or red "X".
